1. Deploy Dengan vLLM

Metode menggunakan library vLLM adalah metode yang paling production ready. Menggunakan vLLM sangat berguna jika proses inference yang ingin dibangun untuk skala produksi yang cukup luas dan tidak hanya untuk konsumsi pribadi. Selain itu, vLLM juga sangat mudah untuk dibangun dan support banyak model LLM yang ada.

Dalam dokumentasi ini, penulis akan memberikan cara untuk melakukan proses inference dengan menggunakan Docker karena lebih efisien dan tidak perlu banyak install software lain.

Pertama, unduh image Docker dari vLLM

xxxxxxxxxx
docker pull vllm/vllm-openai

image tersebut berisikan library vLLM yang akan men-deploy LLM yang dapat diakses melalui rest API dengan format openAI API.

Berikut adalah contoh berkas docker-compose.yaml untuk membuat container Docker untuk deploy model LLM.

xxxxxxxxxx
services:
  vllm:
    image: vllm/vllm-openai:latest # Ini adalah nama image yang tadi di-pull
    container_name: vllm-qwen-server # Nama container yang akan tampil dalam log
    ports:
      - "8000:8000" # Port di mana model akan di-deploy
    volumes:
      - ~/.cache/huggingface:/huggingface # Volume yang wajib di-attach ke container
      - ~/models:/models # Volume berisi model yang telah diunduh
    environment:
      HF_HOME: /huggingface
      HF_TOKEN: <token_akses> # Masukkan token akses yang didapatkan di sini
    ipc: host
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1 # Banyaknya GPU yang ada dalam komputer
              capabilities: [gpu]
    command: >
      /models/Qwen3.5-9B
      --port 8000
      --tensor-parallel-size 1
      --reasoning-parser qwen3
      --enable-auto-tool-choice
      --tool-call-parser qwen3_coder
      --served-model-name Qwen3.5-9B
      --language-model-only
      --enable-prefix-caching
      --speculative-config '{"method": "mtp", "num_speculative_tokens": 2}'
      --reasoning-config '{"reasoning_parser": "qwen3", "reasoning_start_str": "<think>", "reasoning_end_str": "I have to give the solution based on the reasoning directly now.</think>"}'

Untuk penjelasan mengenai parameter-parameter yang ada di dalam vLLM dengan lebih lengkap, bisa kalian lihat dalam tautan berikut.

Buat direktori khusus untuk meletakkan berkas docker-compose.yaml. Misal diletakkan dalam ~/vllm-docker-qwen

xxxxxxxxxx
mkdir -p ~/vllm-docker-qwen

Kemudian, masuk ke dalam direktori tersebut dan deploy container yang sudah diatur dalam docker-compose.yaml

xxxxxxxxxx
cd ~/vllm-docker-qwen;
docker compose up -d;

Untuk memeriksa log dan apakah container dapat berjalan dengan baik, kalian bisa periksa dengan perintah

xxxxxxxxxx
docker compose logs -f

Proses deploy container sudah berhasil jika log menunjukkan keluaran seperti berikut
