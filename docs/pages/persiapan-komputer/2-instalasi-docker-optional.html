2. Instalasi Docker (optional)

Instalasi Docker meskipun opsional, sangat disarankan karena dapat mempermudah dalam proses load dan inference LLM. Dengan meng-install Docker, kita tidak perlu repot untuk meng-install banyak library dan package yang dibutuhkan terutama untuk proses inference LLM. Untuk proses instalasi Docker, kalian dapat lihat dalam dokumentasi resmi dari Docker.

Pertama, kita lakukan set up repositori apt dari Docker

​x
# Add Docker's official GPG key:
sudo apt update;
sudo apt install ca-certificates curl;
sudo install -m 0755 -d /etc/apt/keyrings;
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc;
sudo chmod a+r /etc/apt/keyrings/docker.asc;


# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF;


sudo apt update;

Setelah itu, lakukan proses instalasi pada Docker

xxxxxxxxxx
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y

Setelah instalasi, periksa apakah Docker sudah berjalan dengan

xxxxxxxxxx
sudo systemctl status docker

Jika belum, jalankan secara manual dengan

xxxxxxxxxx
sudo systemctl start docker

Untuk memastikan apakah Docker sudah bisa digunakan, lakukan tes berikut

xxxxxxxxxx
sudo docker run hello-world

Hasil tes akan menunjukkan bahwa container berhasil dibuat dan Docker sudah berjalan dengan baik.

Sebagai tambahan, kita bisa mengatur user kita untuk bisa mengakses docker tanpa menggunakan sudo. Berikut adalah caranya

Pertama, buat group docker. Biasanya otomatis sudah dibuat setelah instalasi Docker

xxxxxxxxxx
sudo groupadd docker

Kemudian, tambahkan user ke dalam group docker

xxxxxxxxxx
sudo usermod -aG docker $USER

Kemudian log out dan log in kembali untuk mengaktifkan user kalian ke dalam group docker. Kalian bisa periksa apakah user kalian sudah masuk ke dalam group docker dengan perintah berikut

xxxxxxxxxx
id -nG $USER

group docker akan termasuk ke dalam keluaran dari perintah tersebut. Dengan begini, kalian bisa menggunakan perintah docker tanpa harus menggunakan sudo

xxxxxxxxxx
docker run hello-world

Setelah meng-install Docker, agar container dapat membaca GPU dalam komputer, Nvidia Container Toolkit harus di-install juga. Pertama, set up repositori apt dari Nvidia Container Toolkit

xxxxxxxxxx
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg \
  && curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
    sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
    sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list;
sudo apt update;

Setelah itu, lakukan proses install Nvidia Container Toolkit

xxxxxxxxxx
sudo apt-get install -y \
  nvidia-container-toolkit \
  nvidia-container-toolkit-base \
  libnvidia-container-tools \
  libnvidia-container1

Setelah selesai meng-install, lakukan konfigurasi pada Docker dengan perintah berikut

xxxxxxxxxx
sudo nvidia-ctk runtime configure --runtime=docker

Perintah di atas akan mengubah isi dari /etc/docker/daemon.json dengan menambahkan field runtime. Terakhir, lakukan restart pada Docker daemon

xxxxxxxxxx
sudo systemctl restart docker

Setelah Docker kembali berjalan, kalian bisa melakukan tes apakah GPU sudah terbaca oleh Docker dengan melakukan perintah berikut

xxxxxxxxxx
sudo docker run --rm --gpus all ubuntu nvidia-smi

Jika keluar hasil dari perintah nvidia-smi, maka Docker telah berhasil membaca GPU dalam komputer.
